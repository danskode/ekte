SKILLeton

Åbent skill-bibliotek

Et bibliotek af skills til ekte. En skill er bare markdown + YAML — det kræver ikke AI at vedligeholde. Det er ekte (med din valgte model) der forbruger dem.

Hvad er en skill?

En markdown-fil med YAML-frontmatter, der tilføjer et system-prompt (en guide) til arbejdet. To begreber der ikke må forveksles:

  • Installation = permanent. Når du installerer en skill, gemmes filen i .ekte/skills/ og bliver liggende.
  • Aktivering = pr. prompt. /skills <navn> injicerer skill'ens ## System Prompt Addition i næste besked og nulstiller bagefter — så en skill aldrig hænger ved utilsigtet.

Sådan indgår det i ekte

ekte henter biblioteket ved runtime fra raw.githubusercontent…/SKILLeton/main/library.yaml:

/skills library # se biblioteket (✓ = installeret) /skills show 3 # læs en skill før install /skills install 1,3 # installér udvalgte /skills bundle aidd # installér en hel pakke

Obligatoriske skills auto-installeres: harness-skills altid (AIDD er præmissen for harnesset), wiki-skills når du tilvælger en wiki.

Pakker (bundles)

Installér en hel kategori på én gang: /skills bundle <navn>.

  • security — security-review, secure-harness, safe-flow, secrets-hygiene
  • ci — ci-hardening, safe-flow
  • aidd — intent-spec-guide, spec-driven-dev, pitch-first, context-layers
  • java · designer — sprog-/domænespecifikke startere
Skill ≠ sensor. En skill er en guide (former hvordan agenten arbejder, før handling). ekte's sensorer er harness-kode (måler resultatet, efter handling). De to er bevidst adskilt.
